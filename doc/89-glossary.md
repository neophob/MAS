# Glossar

**AI-assistierte Dependency Updates**<br>
Unterstützung bei Dependency Updates durch AI-gestützte Analyse, Codeänderungen, Testauswertung
oder Migrationshinweise.<br>

**Dependency Confusion**<br>
Angriff, bei dem ein Paket aus einer öffentlichen Registry gegenüber einem internen Paket bevorzugt
aufgelöst wird.<br>

**Renovate**<br>
Werkzeug zur Automatisierung von Dependency Updates, Pull Requests und Update Policies.<br>

**SBOM**<br>
Software Bill of Materials; strukturierte Auflistung verwendeter Software-Komponenten und
Abhängigkeiten.<br>

**SLSA**<br>
Supply-chain Levels for Software Artifacts; Framework zur Absicherung von Build-Provenance und
Software-Artefakten.<br>

**S2C2F**<br>
Secure Supply Chain Consumption Framework; Referenzrahmen für sicheren Konsum von
Open-Source-Komponenten.<br>

**TTPs**<br>
Tactics, Techniques and Procedures; Beschreibung typischer Vorgehensweisen von Angreifern.<br>
