# AGENTS.md

## Scope

This repository generates the thesis PDF from Markdown in `doc/` using the Node.js tooling in `mdgen/`.
Use Node.js 24 or newer for all local and CI mdgen commands.

## Generate

Run from the repo root:

```bash
cd mdgen
npm run build
```

Output files:

- `mdgen/output/thesis-real.pdf`

## Validate

Run the fast lint checks:

```bash
cd mdgen
npm run lint
```

Run the full local/CI-equivalent validation:

```bash
cd mdgen
npm run test:pixel
```

This rebuilds the Docker image, clears `mdgen/output`, regenerates the PDF, compares it against `template/Thesisvorlage mit Titelbild1.0.pdf`, and writes:

- `mdgen/output/pixel-report.md`
- `mdgen/output/pixel-report.json`
- `mdgen/output/compare/`

For faster local iteration only:

```bash
cd mdgen
MDGEN_PIXEL_USE_CACHE=1 npm run test:pixel
```

## Markdown Inputs

- Authored files live in `doc/` and are processed in filename order.
- Authored images live in `doc/assets/`, including cover/title-page images.
- Special behavior is controlled by frontmatter roles, not filenames.
- `doc/02-toc.md` is generated by `mdgen/src/build.mjs`; do not edit it manually.
- `role: "title"` defines the cover page.
- `role: "abstract"` defines the abstract page.
- Files without a role are numbered thesis chapters.
- Backmatter roles `glossary`, `references`, `appendix`, and `declaration` are rendered unnumbered.

## Notes

- The running footer is generated from the thesis title, current Git hash, and UTC generation date.
- Do not manually edit files in `mdgen/output/`; they are generated artifacts.
